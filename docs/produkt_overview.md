# HeuermannPlus: Übersicht über Features und Anforderungen

HeuermannPlus ist eine Webanwendung zur Organisation von Gruppen und gemeinsamen Aktivitäten. Die Plattform ermöglicht es Benutzern, Gruppen zu erstellen, Mitglieder einzuladen und Aktivitäten zu planen.


## Phase 1: Kernfunktionalitäten

Die folgenden Features sind für die Kernfunktionalität der App vorgesehen:



Registrierung und Anmeldung: Benutzer können sich registrieren und anmelden.

Gruppenverwaltung: Benutzer können Gruppen erstellen und verwalten.

Mitgliederverwaltung: Benutzer können Mitglieder zu Gruppen einladen und verwalten.

Aktivitätenverwaltung: Benutzer können Aktivitäten erstellen und verwalten.

Listendarstellung: Benutzer können Listen von Aktivitäten und Gruppen anzeigen lassen.

AGB-Akzeptanz: Benutzer müssen die AGBs akzeptieren, um die App zu nutzen.


## Phase 2: Erweiterte Funktionen

Die folgenden Features sind für die Erweiterung der App vorgesehen:



Erinnerungen und Benachrichtigungen: Benutzer erhalten Erinnerungen und Benachrichtigungen für Aktivitäten.

Warteliste und Nachrücker: Benutzer können sich für Aktivitäten anmelden und auf eine Warteliste setzen.

Status-Reports: Gruppenverwalter erhalten Status-Reports über die Aktivitäten und Mitglieder.

Mehrsprachigkeit: Die App unterstützt mehrere Sprachen.

Kalender-Integration: Die App kann mit Kalendern integriert werden.

Erweiterte Gruppen- und Rechteverwaltung: Gruppenverwalter können Rechte und Rollen für Mitglieder verwalten.

Protokollierung von Änderungen: Änderungen an der App werden protokolliert.

API für externe Systeme und Frontends: Die App bietet eine API für externe Systeme und Frontends.

Skalierbarkeit und Performance: Die App ist skalierbar und performant.


## Anforderungen

Die folgenden Anforderungen sind für die App vorgesehen:



Benutzerfreundlichkeit: Die App muss benutzerfreundlich sein.

Sicherheit: Die App muss sicher sein und die Daten der Benutzer schützen.

Skalierbarkeit: Die App muss skalierbar sein, um eine große Anzahl von Benutzern zu unterstützen.

Performance: Die App muss performant sein, um eine schnelle und effiziente Nutzung zu ermöglichen.


### Technische Anforderungen

Die folgenden technischen Anforderungen sind für die App vorgesehen:



Webtechnologien: Die App muss auf Webtechnologien wie HTML, CSS und JavaScript basieren.

Datenbank: Die App muss eine Datenbank verwenden, um die Daten der Benutzer zu speichern.

Server: Die App muss auf einem Server gehostet werden, um eine zuverlässige und sichere Nutzung zu ermöglichen.

## Benutzerrollen


Anonymer Anwender: Ein Benutzer, der sich noch nicht registriert hat.

Angemeldeter Anwender: Ein Benutzer, der sich registriert und angemeldet hat.

Mitglied einer Gruppe: Ein angemeldeter Anwender, der Mitglied einer Gruppe ist.

Gruppenverwalter: Ein Benutzer, der eine Gruppe erstellt oder verwaltet.

Administrator (AdmIN): Ein Benutzer mit administrativen Rechten.
